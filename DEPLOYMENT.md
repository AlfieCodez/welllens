# WellLens ‚Äî Deployment & "make it a real app" plan

This covers moving WellLens off the laptop dev server onto a stable HTTPS host, and
the path to getting it onto phones / app stores.

Why it matters:
- A **stable HTTPS domain** so the URL stops changing (no more rotating tunnel links).
- A **public HTTPS endpoint** ‚Äî required for Garmin webhooks (Phase 2) and Google OAuth.
- The **foundation** for installing as a PWA and wrapping for app stores.

---

## 1. Production-readiness changes (before any host)

These are code/config changes we make once, regardless of where we deploy:

| Area | Dev now | Production |
|---|---|---|
| Server | Flask dev server (`run.py`) | **gunicorn** (Linux) or **waitress** (Windows) |
| Debug | `FLASK_DEBUG=1` | `FLASK_DEBUG=0` |
| Admin | `ALLOW_PASSWORD_ADMIN=1` (dev bypass) | **`0`** ‚Äî admin via Google allowlist only |
| Secrets | local `.env` | host secret manager / env vars |
| Cookies | default | `SESSION_COOKIE_SECURE=1`, `HttpOnly`, `SameSite=Lax` |
| Database | SQLite file | SQLite on a **persistent volume**, or **Postgres** if scaling to multiple instances |
| Static files | Flask | **WhiteNoise** (or the platform's CDN) |
| OAuth redirect URIs | `127.0.0.1:5000/...` | `https://yourdomain/...` (Google + Garmin) |

A `ProductionConfig` class, a `Dockerfile`, and a `waitress`/`gunicorn` entrypoint will be
added when we do this step.

## 2. Hosting options (free / cheap, self-hostable)

| Option | HTTPS domain | Cost | Notes |
|---|---|---|---|
| **Fly.io** | yes (`*.fly.dev` + custom) | free small VM | Always-on, good for webhooks. Recommended. |
| **Render** | yes | free tier | Free web service **spins down when idle** (cold starts) ‚Äî okay for testing, weak for webhooks. |
| **Railway** | yes | usage credit | Simple deploys. |
| **VPS (Hetzner ~‚Ç¨4/mo, Oracle free tier)** | yes via **Caddy** (auto-TLS) | cheap/free | Most control; you manage the box. |
| **Named Cloudflare Tunnel + your own domain** | yes, **stable** | free | Points a real domain at a machine you keep running. Free stable URL without a VPS. |

**Recommendation:** **Fly.io** for a proper always-on host (best for Garmin webhooks), or a
**named Cloudflare Tunnel with your own domain** if you want free + keep running it from a
home machine.

## 2a. Deploy to Render (free, no credit card) ‚Äî step by step

Render is the chosen host: a free `https://welllens.onrender.com`-style URL, no card.
Trade-offs on the free plan: it **sleeps after ~15 min idle** (first hit cold-starts in
~40s) and the **filesystem is ephemeral** ‚Äî the SQLite DB resets on each redeploy. That's
fine for testing/demo; for persistent data later, add a paid disk or Postgres, or move to
Oracle Always Free.

Render deploys from a **Git repository**, so the code needs to be on GitHub first.

### Step 1 ‚Äî Put the code on GitHub
You need: a (free) GitHub account, and git. This machine doesn't have git, so either:
- Install **GitHub Desktop** (https://desktop.github.com) ‚Äî includes git, GUI, easiest, or
- Install git CLI (https://git-scm.com/download/win) and run:
  ```powershell
  cd $env:USERPROFILE\Desktop\welllens
  git init
  git add -A
  git commit -m "WellLens"
  git branch -M main
  git remote add origin https://github.com/<you>/welllens.git
  git push -u origin main
  ```
The `.gitignore` already excludes `.env`, `.venv/`, the DB, uploads, and `tools/`, so no
secrets or junk get pushed.

### Step 2 ‚Äî Create the Render service
1. Sign up at https://render.com (GitHub login is fine ‚Äî no card).
2. **New +** ‚Üí **Blueprint** ‚Üí connect your GitHub ‚Üí pick the `welllens` repo.
3. Render reads `render.yaml`, sees the Docker web service, and provisions it on the
   **free** plan. Click **Apply**.

### Step 3 ‚Äî Set the secret env vars
In the service's **Environment** tab, fill the `sync:false` keys:
- `GROQ_API_KEY` ‚Äî your Groq key (AI insights; optional, falls back to templates)
- `ADMIN_EMAILS` ‚Äî your email, for admin
- `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` ‚Äî for Google + admin sign-in (optional)
- `GARMIN_*` ‚Äî only when you have Garmin credentials

`SECRET_KEY` is auto-generated by Render ‚Äî leave it.

### Step 4 ‚Äî Point OAuth/webhooks at the new URL
Once live at `https://<name>.onrender.com`:
- **Google** authorised redirect URIs ‚Üí add
  `https://<name>.onrender.com/auth/google/callback` and `.../admin/auth/google/callback`
- **Garmin** redirect + ping URL ‚Üí your onrender URL (set `GARMIN_REDIRECT_URI` and the
  ping URL with `?token=GARMIN_WEBHOOK_TOKEN`)

### Notes
- **Admin on the public URL:** keep `ALLOW_PASSWORD_ADMIN=0` and use Google sign-in for
  admin. Don't run `seed_dev.py` against production (those are weak dev passwords).
- **Auto-deploy:** every `git push` to `main` redeploys (and resets the free-tier DB).

## 3. Getting onto phones / app stores

WellLens is a **PWA** (done in Phase 3). That gives you installability today. To reach the
stores:

| Target | How | Effort / cost |
|---|---|---|
| **Home-screen install** | PWA "Add to Home Screen" (works now over HTTPS) | none |
| **Google Play** | Wrap the PWA as a **TWA** with PWABuilder / Bubblewrap ‚Üí upload AAB | medium ¬∑ $25 one-time Play fee |
| **Apple App Store** | Wrap with **Capacitor** (WKWebView) + add some native value so review passes | higher ¬∑ $99/yr Apple Developer |

All require the app live on a **public HTTPS domain** first (step 2). The TWA/Capacitor
wrapper just points at that domain.

---

## 4. Suggested order

1. Production config + Dockerfile + waitress/gunicorn (small, one-off).
2. Deploy to Fly.io (or named Cloudflare Tunnel) ‚Üí stable HTTPS domain.
3. Point Google OAuth (and later Garmin) redirect/webhook URLs at the domain.
4. Confirm PWA install from the real domain on your phone.
5. When ready: Android TWA via PWABuilder ‚Üí Play; later iOS via Capacitor.
